## 5️⃣ MOTION & INTERACTION SYSTEM (VERY DETAILED)

### **PHILOSOPHY**
Motion should:
- ✅ Draw attention to important elements
- ✅ Provide feedback (user knows their action registered)
- ✅ Create delight (surprise and joy)
- ✅ Guide the eye (show what to look at next)
- ❌ Never distract or annoy
- ❌ Never be slow (< 300ms for most interactions)

### **MICRO-INTERACTIONS (Individual elements)**

#### 1. **Button Hover States**
```
Resume Download Button:
- Hover: Scale 1.05, glow effect (purple), shadow deepens
- Duration: 200ms ease-out
- Why: Makes clicking feel rewarding
```

#### 2. **Card Hover States**
```
Project/Internship Cards:
- Hover: Scale 1.02, shadow increases, background slight glow
- Duration: 250ms cubic-bezier(0.34, 1.56, 0.64, 1)
- Image inside: Slight brightness increase (10%)
- Why: Indicates clickability, creates anticipation
```

#### 3. **Link Underlines**
```
Skill tags, project links:
- Hover: Underline animates from left to right (100ms)
- Color: Gradient from pink to purple
- Why: Smooth, directional cue
```

#### 4. **Floating Chat Icon** (AI Feature)
```
Chat bubble:
- Default: Pulse animation (scale 1.0 → 1.1 → 1.0, 2s loop)
- Hover: Pulse stops, scale 1.15 with shadow
- Click: Expand to chatbox
- Duration: 400ms on click
- Why: Draws attention without being annoying
```

### **MACRO-INTERACTIONS (Full page/section behaviors)**

#### 1. **Page Load / Landing Section**
```
Sequence (total ~2.5 seconds):
1. Particles appear (0-500ms): Opacity fade-in + subtle scale
2. "Kanishka Sharma" slides in (300-700ms): Transform: translateX(-100px → 0)
3. Tagline fades in (500-900ms): Opacity 0 → 1
4. Intro text appears (700-1200ms): Staggered line reveals
5. Buttons appear (900-1400ms): Scale + fade
6. Scroll indicator appears (1200-1600ms): Subtle pulse

Uses: Staggered animation-delay for each element
Tools: CSS keyframes or GSAP
Why: Creates sense of "system coming to life"
```

#### 2. **Scroll-triggered Section Reveals**
```
As user scrolls to each section:

Skills Section:
- Cards fade in + slide up (from bottom, 300ms delay between each)
- On scroll: Intersection Observer triggers animation
- If graph: Nodes appear one by one, connections animate after

Experience Timeline:
- Each timeline item fades in + slides left (staggered 100ms apart)
- Vertical line animates from top to bottom as items appear
- Why: Guides eye down the timeline naturally

Projects:
- Cards fade in + scale up (staggered)
- Thumbnail images load with gentle blur-to-clear effect
- Why: Feels intentional, not just "everything loaded"
```

#### 3. **Modal Open/Close**
```
Open:
- Backdrop: Opacity 0 → 0.7 (300ms)
- Modal: Scale 0.7 → 1 + opacity 0 → 1 (300ms cubic-bezier for spring feel)
- Content inside: Staggered reveals (100ms apart)
- Why: Feels smooth, not jarring

Close:
- Reverse the animation (200ms, slightly faster)
- Why: Quick acknowledgment of close action
```

#### 4. **Skill Graph / Network Interactions**
```
Network Visualization:
- Initial: Nodes fade in over 1.5s, lines animate after nodes appear
- Hover a node: 
  - Selected node: Scale 1.3, glow
  - Connected nodes: Brighten, others fade to 20% opacity
  - Connected lines: Animate glow effect
  - Duration: 200ms
- Click a node: Show connected projects in tooltip/sidebar

Why: Immediate visual feedback, helps user understand connections
```

#### 5. **Emotion Detection Demo**
```
When user turns on camera:
- Camera feed fades in smoothly (300ms)
- Real-time detection drawing: Lines appear as face detected (fast, < 50ms latency)
- Emotion readout: Numbers fade in + slight color change (green for happy, red for sad, etc.)
- Share screenshot: Button appears with subtle animation

Why: Real-time feedback = engagement gold
```

### **SCROLL BEHAVIOR**

#### 1. **Parallax Effects** (Subtle)
```
Particle background:
- Moves at 20% of scroll speed
- Creates depth feeling
- Not overdone (subtle enough to not distract)

Section titles:
- Slight upward drift as you scroll (5-10% offset)
```

#### 2. **Sticky Navigation**
```
Top nav bar:
- Sticky after first section
- Fade in on scroll down (appears when needed)
- Fades out on scroll up (less clutter)
- Smooth transitions (200ms opacity)

Why: Improves navigation without being intrusive
```

#### 3. **Scroll-to-section Links**
```
Nav click → smooth scroll to section (400ms)
Uses: window.scrollIntoView({ behavior: 'smooth' })
Why: Feels intentional, not jarring
```

### **ANIMATION TOOLS & APPROACH**

#### **For HTML/CSS-heavy:**
- **CSS Keyframes** for simple animations (fade, scale, slide)
- **GSAP** for complex staggered animations and timeline coordination
- **Intersection Observer API** for scroll-triggered animations

#### **For React:**
- **Framer Motion** for smooth component animations
- **React Spring** for physics-based animations
- **Scroll libraries:** Scrollytelling effects with Scroll Trigger

#### **For 3D:**
- **Three.js** for particle system and 3D skill visualization
- **Spline** for pre-made 3D models (easier)

#### **Performance optimization:**
- Use `will-change: transform` sparingly
- GPU acceleration for animations (use `transform` + `opacity`, avoid layout shifts)
- Throttle scroll listeners
- Lazy-load heavy particles on desktop only

### **ANIMATION STYLE GUIDE**

| Interaction | Duration | Easing | Example |
|-----------|----------|--------|---------|
| Hover feedback | 150-250ms | ease-out, cubic-bezier(0.34, 1.56, 0.64, 1) | Button scale |
| Page section reveal | 500-800ms | ease-out | Fade-in text |
| Modal open | 300ms | cubic-bezier(0.34, 1.56, 0.64, 1) | Spring feel |
| Scroll animation | 400-600ms | ease-in-out | Staggered cards |
| Micro-interaction | < 200ms | ease-out | Pulse, underline |
| AI demo feedback | < 100ms | immediate | Real-time emotion |

### **ANIMATION ORCHESTRATION**

Using GSAP timeline for page load:
```javascript
const tl = gsap.timeline();

tl.fromTo('.hero-name', { x: -100, opacity: 0 }, { x: 0, opacity: 1, duration: 1 })
  .fromTo('.hero-tagline', { opacity: 0 }, { opacity: 1, duration: 1 }, '-=0.5')
  .fromTo('.cta-buttons', { y: 20, opacity: 0 }, { y: 0, opacity: 1, duration: 0.8 }, '-=0.5')
  .fromTo('.scroll-indicator', { opacity: 0 }, { opacity: 1, duration: 0.5 });
```

Scroll-triggered animations using Intersection Observer:
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      gsap.fromTo(entry.target, 
        { y: 50, opacity: 0 }, 
        { y: 0, opacity: 1, duration: 0.6 }
      );
    }
  });
});

document.querySelectorAll('.section').forEach(el => observer.observe(el));
```

### **PERFORMANCE OPTIMIZATION**

1. **Code Splitting**
   - Lazy load modal content
   - Code-split AI features (chat, recommender)
   - Dynamic imports for 3D libraries

2. **Image Optimization**
   - Use WebP format with fallbacks
   - Lazy load project screenshots
   - Compress certificate images
   - Use blur-up placeholder effect

3. **Animation Performance**
   - Use `will-change` sparingly
   - GPU acceleration (transform, opacity only)
   - Reduce particle count on mobile
   - Throttle scroll listeners

4. **Bundle Size**
   - Tree-shake unused libraries
   - Use lightweight alternatives (date-fns instead of moment)
   - Don't ship polyfills users don't need

5. **Metrics to Track**
   - LCP (Largest Contentful Paint): Target < 2.5s
   - FID (First Input Delay): Target < 100ms
   - CLS (Cumulative Layout Shift): Target < 0.1
